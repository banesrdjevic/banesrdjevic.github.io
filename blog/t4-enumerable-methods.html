Let's take a look at Enumerable#cycle today. Cycle is a method you can call on any enumerable (obviously), and as you might have guessed it cycles through the elements of the enumerable.

Here's the syntax:

Enumerable.cycle(number){|element in the enumerator| code}

What this does is cycle through each part of the enumerator as many times as indicated by "number" and performs the specified code on each element. That means:

array.cycle(6){|x| puts x}

will puts each element of the array in order for a total of six times. Simple enough, but there are two things that you absolutely need to watch out for.

THINGS TO WATCH OUT FOR

First:

If you do not include (number), like so:

array.cycle{|x| puts x}

you will create an infinite loop and break your code! Not adding (number) tells the program to cycle through the array an infinite amount of times. Do not do this unless you have a reason for it!

Second:

Cycle is a non-destructive method, which means that it will never alter the value of the enumerable permanently. It alters it temporarily each time it iterates over the element, but it reverts as soon as the iteration switches to a new element. That's why:

array = [1,2,3]

array.cycle(3){|x| x=x*x}

print Array

will output [1,2,3] instead of [1, 256, 6561] . It goes through the array, but it doesn't change it.

But what is this good for, then?

What's the Point?

Well, while cycle doesn't change the enumerable it's being used on, it can be used on one enumerable to change a whole bunch of others! We could do this:

array=[1,2,3]

array_2=[]

array.cycle(1){|x| array_2<<(x*x)}

and there you go! It's a very simple usage, but it can be incredibly useful in a pinch or two.

So in closing, make sure you don't go to infinity and beyond, realize that no matter how hard you try you won't have the power to change the things you want to control most, and use this method to get what you want from others. Simple and Clean, like Utada Hikaru.
